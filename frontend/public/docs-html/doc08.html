Visao Geral

O Sistema de Autenticacao da Nuvary Invest implementa um fluxo completo de registro, login e recuperacao de senha utilizando JWT (JSON Web Tokens) para gerenciamento de sessoes. O sistema foi projetado seguindo as melhores praticas de seguranca e conformidade com a LGPD.

Funcionalidades

  • Cadastro - Registro com validacao de campos e aceite de termos
  • Login - Autenticacao com JWT (access + refresh tokens)
  • Logout - Revogacao de tokens
  • Recuperacao de Senha - Fluxo completo com token temporario
  • Perfil de Investidor - Vinculacao do questionario ao usuario
  • Historico de Chat - Persistencia das conversas

Validacao com Zod

// Schema de cadastro const registerSchema = z.object({ nome: z.string().min(3, 'Nome deve ter pelo menos 3 caracteres'), email: z.string().email('Email invalido'), senha: z.string() .min(8, 'Minimo 8 caracteres') .regex(/[a-z]/, 'Deve conter minuscula') .regex(/[A-Z]/, 'Deve conter maiuscula') .regex(/[0-9]/, 'Deve conter numero'), confirmarSenha: z.string(), aceiteTermos: z.boolean().refine(val => val === true), }).refine(data => data.senha === data.confirmarSenha, { message: 'Senhas nao conferem', path: ['confirmarSenha'], });

Seguranca

Hash de Senhas

Senhas sao hasheadas com bcrypt (10 salt rounds) antes de armazenar. Nunca sao salvas em texto plano.

JWT com Refresh Token

Access token expira em 1 hora. Refresh token expira em 7 dias e pode ser revogado no logout.

Rate Limiting

Maximo 5 tentativas por minuto nas rotas de auth para prevenir brute force.

Headers de Seguranca

Helmet configura headers HTTP como X-Content-Type-Options, X-Frame-Options, etc.

Configuracao JWT

Parametro Valor
Algoritmo HS256
Access Token Expiracao 1 hora
Refresh Token Expiracao 7 dias
Payload userId, email, nome

Validacao de Senha

  • Minimo 8 caracteres
  • Pelo menos 1 letra maiuscula
  • Pelo menos 1 letra minuscula
  • Pelo menos 1 numero

Conformidade LGPD

  • Consentimento explicito obrigatorio (checkbox de termos)
  • Data/hora do aceite armazenada
  • Direito ao esquecimento (DELETE /api/auth/account)
  • Dados sensiveis nunca expostos em logs ou respostas

Fluxos de Autenticacao

Fluxo de Cadastro

Formulario → Validacao Zod → POST /register → Hash bcrypt → Salvar DB → Redireciona Login

Fluxo de Login

Credenciais → POST /login → Verifica bcrypt → Gera JWT → Salva localStorage → Redireciona App

Fluxo de Refresh Token

Token Expirado → POST /refresh → Valida Refresh → Novo Access Token → Continua Request

Fluxo de Recuperacao de Senha

Email → Gera Token → Envia Email* → Token + Nova Senha → Atualiza DB

* Envio de email nao implementado - token exibido no console para testes.
