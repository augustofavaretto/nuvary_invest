Funcionalidades

1. Aba Dados Pessoais

  • Visualizar nome: Exibe o nome completo do usuario
  • Editar nome: Modal para alterar o nome com validacao
  • Visualizar email: Exibe o email atual (somente leitura)
  • Data de criacao: Mostra quando a conta foi criada

2. Aba Perfil de Investidor

  • Tipo de perfil: Conservador, Moderado ou Agressivo
  • Data do questionario: Quando o perfil foi definido
  • Descricao do perfil: Caracteristicas do tipo de investidor
  • Refazer questionario: Botao para atualizar o perfil
Perfis de Investidor:
  • Conservador: Prioriza seguranca, baixa tolerancia a risco
  • Moderado: Equilibrio entre seguranca e rentabilidade
  • Agressivo: Busca maiores retornos, aceita mais risco

3. Aba Seguranca

  • Alterar senha: Requer senha atual + nova senha + confirmacao
  • Alterar email: Envia confirmacao para novo email
  • Logout: Encerra a sessao atual
Importante: Senhas sao armazenadas com hash (bcrypt) no Supabase Auth. Nao e possivel visualizar a senha atual, apenas alterá-la.

// Atualizar email
export async function atualizarEmail(novoEmail: string) {
  // Envia email de confirmacao para o novo endereco
  const { error } = await supabase.auth.updateUser({
    email: novoEmail
  });
}

// Atualizar senha
export async function atualizarSenha(
  senhaAtual: string,
  novaSenha: string
) {
  // 1. Verifica senha atual fazendo login
  const { error: loginError } = await supabase.auth.signInWithPassword({
    email: user.email,
    password: senhaAtual
  });

  if (loginError) throw new Error('Senha atual incorreta');

  // 2. Atualiza para nova senha
  const { error } = await supabase.auth.updateUser({
    password: novaSenha
  });
}

Validacoes de Senha

  • Minimo 8 caracteres
  • Pelo menos uma letra maiuscula
  • Pelo menos uma letra minuscula
  • Pelo menos um numero
  • Confirmacao deve ser igual

Consideracoes de Seguranca

Protecao de Dados

  • Senhas: Hash com bcrypt (Supabase Auth nativo)
  • Sessao: JWT com refresh token automatico
  • RLS: Usuarios so acessam seus proprios dados
  • Validacao: Senha atual obrigatoria para alteracoes sensiveis
