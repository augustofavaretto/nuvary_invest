🎯 Contexto e Problema

Dois problemas críticos foram resolvidos nesta versão:

SQLite efêmero no Railway — o arquivo nuvary.db era apagado a cada redeploy do backend, perdendo todos os dados de usuários e histórico de chat.
Carteira isolada no localStorage — ativos ficavam presos no browser local, sem sincronização entre dispositivos e sem vínculo com o usuário autenticado.
Solução — migração completa para Supabase PostgreSQL com RLS (Row Level Security), garantindo persistência permanente e dados por usuário.

⚡ Trigger de Auto-Criação de Perfil

A tabela profiles é populada automaticamente quando um novo usuário faz signup no Supabase Auth, via trigger PostgreSQL:

CREATE OR REPLACE FUNCTION public.handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO public.profiles (id, nome, email, aceite_termos) VALUES ( NEW.id, COALESCE(NEW.raw_user_meta_data->>'nome', NEW.email), NEW.email, FALSE ) ON CONFLICT (id) DO NOTHING; RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
O campo nome é preenchido automaticamente a partir de raw_user_meta_data — que inclui o nome passado no signup. Se não houver, usa o email como fallback.
