🔧 Correcao do Fluxo de Reset de Senha

Problema identificado

O onAuthStateChange no ResetPasswordForm conflitava com o AuthContext: quando logout() era chamado apos o reset, o evento SIGNED_OUT re-renderizava o componente para "Link expirado" antes de setIsSuccess(true) ser processado pelo React.

Solucao implementada

  • ✅Removido onAuthStateChange do ResetPasswordForm — conflitava com o listener do AuthContext
  • ✅Sessao validada apenas uma vez com getSession() no mount do componente
  • ✅Substituido router.push('/login') por window.location.href = '/login' — hard redirect garante estado limpo
  • ✅Chamada direta supabase.auth.signOut() em vez de logout() do service — evita side effects

Fluxo de reset corrigido

  1. 1 Usuario clica em "Esqueci minha senha" → digita email → Supabase envia email com link
  2. 2 Link do email redireciona para /redefinir-senha com token no fragment da URL
  3. 3 AuthContext detecta evento PASSWORD_RECOVERY → redireciona para /redefinir-senha
  4. 4 ResetPasswordForm monta → getSession() valida sessao → exibe formulario
  5. 5 Usuario digita nova senha → supabase.auth.updateUser({ password }) → senha salva
  6. 6 supabase.auth.signOut() → tela de sucesso → window.location.href = '/login' apos 2s

Codigo final do submit

const onSubmit = async (data: ResetFormData) => {
  try {
    await redefinirSenha(data.novaSenha); // supabase.auth.updateUser
  } catch (error) {
    setServerError('Sessao expirada. Solicite um novo link.');
    return;
  }

  try { await supabase.auth.signOut(); } catch { /* ignora */ }

  setIsSuccess(true);
  setTimeout(() => {
    window.location.href = '/login'; // hard redirect
  }, 2000);
};
